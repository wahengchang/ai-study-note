## Execution Protocol (per `cron/git-auto.md`)

For each actionable issue, the cron runner must:

1. Mark `in_progress` in `.automation/issues.json` before starting
2. Branch fresh: `git fetch origin && git checkout -B auto/issue-<N> origin/main`
3. Delegate to `@writer` agent with the issue body as input
4. Output: one Markdown file under `content/` following project conventions
5. Validate: `npm run quartz -- build` exits 0
6. Stage by explicit path only — never `git add .`
7. Commit, push, create one PR per issue
8. Mark `pr_open` on success, `failed` on build failure
