## Execution Protocol

Per `cron/git-auto.md`:

1. Process one issue per run, lowest number first
2. Branch from fresh `origin/main`: `git checkout -B auto/issue-<N> origin/main`
3. `@writer` drafts the note following `claude/agents/writer.md` output format
4. `@diagram` generates Mermaid diagrams where marked as support agent
5. `@reviewer` performs QA pass before PR creation
6. Validate with `npm run quartz -- build`
7. Stage by explicit path, commit, push, open PR
8. Update `.automation/issues.json` tracker (local only, never committed)
