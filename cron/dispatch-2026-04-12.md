## Execution Protocol

Per `cron/git-auto.md`:

1. Each run processes **one issue** (lowest number first)
2. Branch from fresh `origin/main`: `git checkout -B auto/issue-<N> origin/main`
3. `@writer` agent composes the deliverable using `formatting.md` + `mermaid.md` + `quartz.md`
4. Validate: `npm run quartz -- build` exits 0
5. Commit by explicit path, push, open PR
6. Tracker updated to `pr_open` on success, `failed` on validation failure
7. Next run picks the next pending issue
