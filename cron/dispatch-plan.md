## Execution Protocol

Each run follows `cron/git-auto.md` invariants:

1. Preflight: clean working tree, fetch origin
2. Load `.automation/issues.json`, pick lowest pending issue
3. Mark `in_progress` in tracker
4. Branch: `git checkout -B auto/issue-<N> origin/main`
5. Delegate to `@writer` with issue body as context
6. Validate: `npm run quartz -- build` exits 0
7. Commit by explicit path, push, create PR referencing the issue
8. Mark `pr_open` in tracker
9. Stop — one issue per run
