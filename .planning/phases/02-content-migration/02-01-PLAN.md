    Step 1 — Install dev dependency:

    ```bash
    npm install --save-dev gray-matter
    ```
    Expected result: `package.json` `devDependencies` now contains `"gray-matter": "^<version>"`. No runtime deps added.

    Step 2 — Create `scripts/migrate-content.mjs` as a self-contained Node ESM script (the repo is `"type": "module"` per package.json, so `.mjs` is belt-and-suspenders explicit). The script MUST be idempotent: running it twice on a clean tree produces identical output.

    Script structure (implement each numbered block as a function; wire them in `main()`):

    1. **Paths (hardcode absolute via `path.resolve(process.cwd(), …)`):**
       - `SRC_ROOT = "content"`
       - `DEST_ROOT = "src/content/blog"`
       - `ASSET_SRC = "content/assets"`
       - `ASSET_DEST = "public/assets"`
       - `ASSET_URL_PREFIX = "/ai-study-note/assets/"` (base-prefixed absolute path — Astro's `base: "/ai-study-note"` means published URLs use this prefix; we write the final URL into the Markdown body because Markdown body can't use `import.meta.env.BASE_URL`)

    2. **Walk `content/` recursively** using `fs.readdirSync(..., { recursive: true, withFileTypes: true })`. For each entry:
       - Skip directories matching `.obsidian` (any depth)
       - Skip anything under `content/assets/` (handled by the asset copy phase)
       - Skip non-`.md` files
       - Skip files whose basename starts with `.` or `._` (macOS resource forks — the repo has many)
       Collect into an array of `{ absPath, relPath, topCategory, basename }` where `topCategory = relPath.split(path.sep)[0].toLowerCase()` (for `content/index.md` at the root, set `topCategory = "index"`).

    3. **Derive frontmatter for each file** using `gray-matter`:
       - `const parsed = matter(fs.readFileSync(absPath, "utf8"));`
       - `const fm = parsed.data;`
       - `const body = parsed.content;`

       Build the new frontmatter object in this exact order:
       - `title`: `fm.title ?? humanize(basename)` where `humanize("ch1-architecture-four-pillars") → "Ch1 Architecture Four Pillars"` (split on `-`, capitalize each word, join with space)
       - `description`: `fm.description ?? firstParagraph(body) ?? title` where `firstParagraph` returns the first non-heading, non-blank line block, collapsed to a single line, trimmed to 160 chars with ellipsis if truncated. Skip lines starting with `#`, `-`, `*`, `>`, `!`, `:`, `|`, or fenced code blocks.
       - `pubDate`: read via `git log --diff-filter=A --format=%aI -- <relPath> | tail -1`. If the command returns empty (file never committed, or git not available), fall back to `new Date().toISOString().slice(0, 10)` (today as `YYYY-MM-DD`). Implement with `execSync` and try/catch; on ANY error fall through to today.
       - `category`: `topCategory` (already lowercased). For the root `content/index.md`, use `"index"`.
       - `tags`: `Array.isArray(fm.tags) ? fm.tags : (typeof fm.tags === "string" ? [fm.tags] : [])` — do NOT normalize case here; the schema does it at load time.
       - `draft`: `fm.draft === true ? true : false`

    4. **Transform the body** in this exact order (each regex operates on the full string):
       a. Image wikilinks (greediest first): `/!\[\[([^\]]+?)\]\]/g` → `"![]" + "(" + ASSET_URL_PREFIX + $1 + ")"`. Note: the captured path may include a subdirectory (e.g. `openclaw/foo.png`); preserve it verbatim.
       b. Labeled wikilinks: `/\[\[([^\]|]+?)\|([^\]]+?)\]\]/g` → `$2` (keep the label, drop the link)
       c. Bare wikilinks: `/\[\[([^\]]+?)\]\]/g` → last path segment of `$1` (e.g. `openclaw/ch1-architecture-four-pillars` → `ch1-architecture-four-pillars`) so the visible text is reasonable
       d. Smart Columns: split body into lines, drop any line whose trim matches `/^:::col(\b|\{|$)/` (opening fence with or without attributes) or `/^:::$/` (closing fence). Keep EVERYTHING between the fences as-is.
       e. Re-join lines with `\n`

    5. **Determine destination filename (flat layout):**
       - Primary: `<basename>` (e.g. `ch1-architecture-four-pillars.md`)
       - If another already-processed file claimed the same basename, prefix with `<topCategory>-<basename>` (e.g. `openclaw-index.md`). Track claimed basenames in a `Set`.
       - The root-level `content/index.md` becomes `index.md` (first claim wins; it will claim before any `<folder>/index.md`).
       - Second-collision safety: if even the category-prefixed name is taken, append `-<parentDirName>` (e.g. `openclaw-common-questions-index.md`). Log each collision to stderr.

    6. **Write output:**
       - `fs.mkdirSync(DEST_ROOT, { recursive: true });`
       - `fs.writeFileSync(path.join(DEST_ROOT, destName), matter.stringify(transformedBody, newFrontmatter));`
       - gray-matter's `stringify` produces deterministic YAML — acceptable for idempotency.

    7. **Copy assets** AFTER markdown migration:
       - `fs.cpSync(ASSET_SRC, ASSET_DEST, { recursive: true, force: true });` (Node 22+ has `cpSync` with recursive)
       - This preserves subdirectory structure (e.g. `content/assets/openclaw/foo.png` → `public/assets/openclaw/foo.png`).
       - Filter: skip macOS `._*` files during copy (use a `filter` callback).

    8. **Summary report** to stdout at the end:
       ```
       Migration complete:
         Files scanned:      <N>
         Files migrated:     <N>
         Basename collisions: <N>  (with list)
         Wikilinks stripped:  <N>  (image + labeled + bare)
         Smart Columns removed: <N>  (fence pairs)
         Assets copied:       <N>
         Files falling back to today's date: <N>
       ```
       Exit 0 on success; exit 1 if ANY file produced zero-length output or if gray-matter throws.

    Step 3 — Do NOT execute the script in this task. The script's existence + dependency install is the deliverable. Execution is Task 2.

    Why this shape (rationale captured for the checker):
    - Single script keeps the transformation auditable in one place (per 02-CONTEXT.md Specifics)
    - Idempotent because gray-matter stringify is deterministic and we always rewrite from source
    - Base-prefixed absolute URLs in Markdown bodies work because Astro's `base: "/ai-study-note"` is already applied at the URL level; `public/assets/` is served at `${base}/assets/`
    - Skipping `.obsidian` and `._*` avoids pulling in Obsidian vault metadata and macOS resource forks
