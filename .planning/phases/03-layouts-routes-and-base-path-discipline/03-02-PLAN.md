    Create `src/pages/blog/[...slug].astro` with the code below. This is the static-dynamic route that generates one page per blog post at build time.

    ```astro
    ---
    import { getCollection, render, type CollectionEntry } from "astro:content";
    import PostLayout from "../../layouts/PostLayout.astro";

    export async function getStaticPaths() {
      const posts = await getCollection("blog", ({ data }) => !data.draft);
      return posts.map((post) => ({
        params: { slug: post.id },
        props: { post },
      }));
    }

    interface Props {
      post: CollectionEntry<"blog">;
    }
    const { post } = Astro.props;
    const { Content } = await render(post);
    ---
    <PostLayout post={post}>
      <Content />
    </PostLayout>
    ```

    Rationale:
    - `getStaticPaths` is MANDATORY for `[...slug]` routes in static output mode. It runs at build time and returns one `{ params, props }` per post.
    - `params.slug = post.id` â€” `post.id` is the filename without extension (from the `glob` loader in `content.config.ts`). This produces URLs like `/ai-study-note/blog/agent-teams-guide/`.
    - `render(post)` is the Astro 6 pattern (imported from `astro:content`) â€” returns `{ Content }`, a component that renders the Markdown body through the remark/rehype pipeline.
    - Draft filter again: `({ data }) => !data.draft` â€” even at the `getStaticPaths` level. Draft posts will not generate pages. Satisfies ROUT-07.
    - Passing `post` as a prop lets `PostLayout` render the title/description/meta without re-querying the collection.
    - After this task, run `npm run build`. The build output should contain one HTML file under `dist/blog/<slug>/index.html` for every non-draft post (72 expected).
  </action>
