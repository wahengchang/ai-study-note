# The 3 Core Markets Framework — 市場選擇、利基定位與產品構思

> 來源：Russell Brunson《Expert Secrets》、Custom GPT "The 3 Core Markets Market Ideas Generator"、多方網路研究整理

## TL;DR

所有可獲利的市場都歸屬於三大核心市場：**Health（健康）**、**Wealth（財富）**、**Relationships（人際關係）**。成功的利基定位不是「選一個大市場」，而是沿著 **Core Market → Sub-market → Niche → Micro-niche** 的階層往下鑽，找到一個你能成為領導者的小領域，然後建立「新機會（New Opportunity）」而非「改良型產品（Improvement Offer）」。

---

## 1. 核心框架：三大市場與階層結構

### 1.1 三大核心市場

| Core Market | 核心慾望 | 人們購買的本質 |
|-------------|----------|----------------|
| **Health（健康）** | 身體更好、活更久、更有精力、外表更好 | 減肥、健身、營養、心理健康、睡眠、抗老化 |
| **Wealth（財富）** | 賺更多錢、財務自由、職涯成長 | 投資、副業、創業、被動收入、省錢、理財 |
| **Relationships（人際關係）** | 更好的連結、愛情、家庭、社交 | 約會、婚姻修復、育兒、社交技巧、自我成長 |

> 每一筆消費都是為了改善這三個領域之一。如果你的產品不歸屬於其中之一，市場規模可能不夠大。

### 1.2 四層階層結構

```
Core Market (3個)
  └── Sub-market (數十個)
        └── Niche (數百個)
              └── Micro-niche (無限個)
```

**實例演示：**

| 層級 | Health 範例 | Wealth 範例 | Relationships 範例 |
|------|------------|------------|-------------------|
| Core Market | Health | Wealth | Relationships |
| Sub-market | 減重 | 線上創業 | 約會 |
| Niche | 女性產後減重 | Shopify 電商 | 40 歲以上男性重新約會 |
| Micro-niche | 用間歇性斷食的哺乳媽媽減重 | 用 AI 自動化的 Shopify POD 店 | 離婚後的內向男性用交友 App |

## 3. 市場選擇與驗證邏輯

### 3.1 五步選擇流程

```
Step 1: 你屬於哪個 Core Market？ (Health / Wealth / Relationships)
         ↓
Step 2: 列出你有經驗/熱情/專業的 Sub-markets
         ↓
Step 3: 在 Sub-market 中找到 Niche（組合人群 + 方法 + 結果）
         ↓
Step 4: 驗證需求（搜尋量、社群活躍度、競品存在）
         ↓
Step 5: 確認你能成為該 Niche 的「唯一」或「最好」之一
```

### 3.2 Niche 組合公式

```
Niche = [特定人群] + [特定方法/工具] + [特定結果]
```

範例：
- `哺乳媽媽` + `間歇性斷食` + `產後恢復體重` = Micro-niche
- `台灣自由接案者` + `Claude AI` + `提案效率翻倍` = Micro-niche
- `離婚後的 40 歲男性` + `交友 App 策略` + `重建約會信心` = Micro-niche

### 3.3 驗證清單（Decision Criteria）

在選定 Niche 前，逐項確認：

| 驗證項目 | 通過標準 | 檢查方法 |
|---------|---------|---------|
| **市場有熱情的粉絲嗎？** | 有活躍社群、論壇、FB 社團 | 搜尋 Reddit、FB Groups、Discord |
| **有既有競品嗎？** | 有 = 好事（驗證需求存在）| 搜尋 Google、Udemy、Gumroad |
| **有行業用語嗎？** | 有專屬術語 = 深度社群 | 潛入論壇觀察 |
| **有活動/會議嗎？** | 有線上線下活動 = 高參與度 | 搜尋 Eventbrite、Meetup |
| **有名人/KOL 嗎？** | 有 = Sub-market 已被驗證 | 搜尋 YouTube、Podcast |
| **你有獨特的視角嗎？** | 你的經歷、框架、組合有差異化 | 自我評估 |
| **能提供 New Opportunity 嗎？** | 不是「更好的 X」而是「全新的方式」| 框架測試 |

### 3.4 Red Ocean vs. Blue Ocean 判斷

Russell Brunson 的核心觀點：**不要在紅海裡競爭，也不要去無人的藍海。你要做的是在紅海旁邊創建自己的藍海。**

- **紅海（避免）**：直接在已有領導者的 Niche 裡競爭（例：直接做「減肥」）
- **藍海陷阱（避免）**：完全沒有競品的市場（可能代表沒有需求）
- **正確做法**：在成熟的 Sub-market 旁邊，用新角度、新方法、新組合創建自己的 Niche

## 4. 構思與腦力激盪 Prompts

以下是可直接複製使用的 AI Prompt，用於不同階段的市場研究。

### 4.1 Niche 發想 Prompt

```
你是一位市場分割專家。請在「{Core Market}」核心市場下，用以下階層結構生成利基點子：

Core Market
  └── Category（盡可能多）
        └── Sub-category（盡可能多）
              └── Niche（盡可能多）
                    └── Micro-niche（盡可能多）

聚焦在 "{Sub-market}" 這個子市場。
確保每個層級的項目不重疊、各自獨特。
用繁體中文回覆，但專有名詞保留英文。
```

### 4.2 Reddit 痛點挖掘 Prompt

先用以下 Google 搜尋語法找 Reddit 討論：

```
"{你的市場關鍵字}" (
   site:reddit.com
   inurl:comments|inurl:thread
   | intext:"I think"|"I feel"|"I was"|"my biggest struggle"|
   "my biggest fear"|"I found that"|"struggles"|"problems"|
   "pain point"|"frustrations"|"what I wish I knew"
)
```

再將找到的 Reddit 內容貼給 AI，搭配以下 Prompt：

```
你是一位市場研究分析師，專長分析對話資料以找出痛點和未被滿足的需求。

請分析以下 Reddit 討論，並：
1. 找出所有痛點，按主題分類
2. 每個痛點附上 3-5 則原文引用
3. 評估每個痛點的頻率和強度
4. 按照「頻率 × 強度 × 可解決性」排序

只提取具體問題、對現有方案的不滿、和未被滿足的需求。
忽略一般性討論和正面體驗。

[在此貼上 Reddit 內容]
```

### 4.3 市場缺口分析 Prompt

```
根據以下已識別的痛點，請用五個框架生成商業機會：

1. 市場分割框架：找出被忽視的子族群
2. 產品差異化框架：高級版 / 精簡版 / 專業版
3. 商業模式創新：訂閱制 / 免費增值 / 平台模式
4. 通路與行銷框架：未被充分利用的獲客管道
5. 新範式框架：新技術、新趨勢的應用

每個方案需包含：名稱、說明、目標受眾、差異化、商業模式、如何解決痛點。
最後用「市場規模 × 競爭優勢持續性 × 可行性」排出 Top 3。

痛點清單：
[在此貼上痛點分析結果]
```

### 4.4 產品/內容構思 Prompt

```
我選定的利基是：[描述你的 Micro-niche]

請用以下角度各生成 5 個產品/內容點子：
1. 數位課程（解決核心痛點的系統化教學）
2. 模板/工具（省時省力的實用資源）
3. 社群/會員制（持續性價值）
4. 1-on-1 服務（高價值客製化）
5. 免費引流內容（吸引流量的 Blog/YouTube 主題）

每個點子附上：標題、一句話價值主張、定價建議、目標客戶描述。
```

## 5. 工作流程：Research → Validate → Position → Launch

### Phase 1: Research（研究）

| 動作 | 工具 | 產出 |
|------|------|------|
| 確認 Core Market | 自我評估 | 選定 1 個核心市場 |
| 列出 Sub-markets | AI 腦力激盪 + 市場觀察 | 5-10 個候選子市場 |
| 挖掘痛點 | Reddit + Google 搜尋 + AI 分析 | 痛點清單（按優先級排序） |
| 分析競品 | Google、Udemy、Gumroad、YouTube | 競品地圖：誰在做什麼、缺什麼 |

### Phase 2: Validate（驗證）

| 動作 | 工具 | 通過門檻 |
|------|------|---------|
| 搜尋量驗證 | Google Trends、Ubersuggest、Ahrefs | 有穩定或上升趨勢 |
| 社群活躍度 | Reddit、FB Groups、Discord | 月活帖數 > 50 |
| 付費意願 | Gumroad、Udemy、Amazon 暢銷書 | 有人在賣類似產品且有銷量 |
| MVP 測試 | Landing page + 預購/Email 註冊 | 轉換率 > 2% |

### Phase 3: Position（定位）

| 動作 | 說明 | 產出 |
|------|------|------|
| 定義 New Opportunity | 不是「更好的 X」，而是「用 Y 方式達成 Z」| 一句話定位語 |
| 建立 Origin Story | 你為什麼是對的人來教這件事？ | 個人故事框架 |
| 差異化聲明 | 你跟競品的三個關鍵差異 | 差異化清單 |
| 目標受眾人物誌 | 具體描述你的理想客戶 | 1 頁 Persona 文件 |

### Phase 4: Launch（上市）

| 動作 | 內容 | 目標 |
|------|------|------|
| 引流內容 | 5-10 篇解決痛點的免費內容 | 建立信任 + Email List |
| Lead Magnet | 免費資源（checklist、模板、mini-course）| 收集 Email |
| 銷售漏斗 | Landing page → Email sequence → Offer | 轉換為付費客戶 |
| 社群建立 | Discord/FB Group/LINE 社群 | 留存 + 口碑 |

## 6. 可複用原則

### 原則 1：永遠從 Core Market 往下鑽，不要從 Niche 往上猜
先確認你在三大市場的哪一個，再系統性地細分。跳過這步容易選到沒有市場的利基。

### 原則 2：有競品是好事，沒競品才可怕
競品存在 = 需求被驗證。你要做的不是進入無人區，而是在已驗證的市場旁邊創建自己的新類別。

### 原則 3：賣 New Opportunity，不是賣 Improvement
「教你更好的減肥方法」是改良型（紅海）。「用 AI 為你的基因型量身打造飲食計畫」是新機會（藍海）。

### 原則 4：Niche = 人群 + 方法 + 結果
不要只選一個主題。明確的人群 + 獨特的方法 + 具體的結果 = 不可替代的定位。

### 原則 5：用痛點語言，不是用你的語言
用 Reddit 和社群裡的真實用語來描述問題和撰寫行銷文案，而非自己發明的術語。

### 原則 6：先驗證再建造
Landing page + Email signup 的成本接近零。在花三個月做產品之前，先測試有沒有人願意留下 Email。

### 原則 7：能被 AI 加速的 Niche 有時間優勢
選擇能用 AI 工具（Claude、n8n、自動化）加速研究、內容生產、客戶互動的 Niche，你的產出速度就是競爭壁壘。

## 7. Quick Reference（速查表）

### 三大市場速查

```
Health（健康）→ 減重 / 健身 / 營養 / 心理健康 / 睡眠 / 抗老化 / 替代療法
Wealth（財富）→ 創業 / 投資 / 被動收入 / 接案 / 職涯 / 理財 / AI 自動化
Relationships（關係）→ 約會 / 婚姻 / 育兒 / 社交 / 自我成長 / 寵物
```

### Niche 定位公式

```
[特定人群] + [特定方法/工具] + [特定結果] = 你的 Micro-niche
```

### 驗證三問

```
1. 有人在 Reddit/FB 討論這個問題嗎？ → 需求存在
2. 有人在賣解決方案嗎？               → 付費意願存在
3. 我能用不同的角度切入嗎？           → 差異化可能
```

### 工作流一句話

```
Research（挖痛點）→ Validate（測需求）→ Position（建定位）→ Launch（做產品）
```

### 決策矩陣（快速評分）

針對每個候選 Niche，用 1-5 分評估：

| 評分項目 | 權重 | 你的分數 |
|---------|------|---------|
| 市場需求（搜尋量、社群活躍度） | 30% | ___ |
| 你的專業/熱情/經驗 | 25% | ___ |
| 競爭程度（有競品但不擁擠） | 20% | ___ |
| 獲利潛力（付費意願、客單價） | 15% | ___ |
| AI 加速可能性 | 10% | ___ |
| **加權總分** | 100% | ___ |

> 選加權總分最高的 2-3 個做 MVP 測試，而非只選一個。
